# 생성 지시: 대화 수리 (Repair)

이전에 생성된 은행 챗봇 대화가 품질 검증을 통과하지 못했다.
아래 **검출된 문제 목록**을 모두 고쳐서, 동일한 의도·라벨·상태를 유지한 채 대화를 다시 작성하라.

## 원래 생성 지시 (그대로 지켜야 함)
{ORIGINAL_INSTRUCTION}

## 검출된 문제 목록 (반드시 모두 해결)
{ISSUES}

## 이전 생성 결과 (수정 대상)
```json
{PREVIOUS_OUTPUT}
```

## 수리 규칙
- 위 문제를 **모두** 해결하라.
- 원래 의도한 generation 유형, 표적 라벨, 사건 상태(occurred/upcoming/weak/cancelled/no_event)는 바꾸지 마라.
- 턴 수/사용자 발화 수 규칙(단일 세션 7~10턴, 사용자 4~6턴)을 다시 맞춰라.
- 이모지·초성체·FA 코드·헤더·직접 라벨·챗봇 요약 leakage를 제거하라.
- 출력은 원래 지시와 **동일한 JSON 형식**으로만 작성하라. 설명 문장 금지.
