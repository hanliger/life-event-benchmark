# Generation Guidelines

업로드된 Life Event Pool의 생성 가이드를 pilot용으로 압축한 문서입니다.

## User persona

- 사용자는 자기 상황을 설명하러 온 사람이 아니라 당장 처리할 금융 업무가 있어서 온 사람이다.
- Life Event를 직접 말하지 않는다.
- 단서는 금융 업무를 처리하는 과정에서 자연스럽게 드러난다.
- 사용자 발화는 짧고 구어체이며, 문장부호가 적고 가벼운 맞춤법 오류가 있을 수 있다.
- 초성체와 이모지는 사용하지 않는다.

## Cue channels

Medium 난이도에서는 단서를 서로 다른 채널에 분산한다.

| Channel | Meaning | Example |
|---|---|---|
| 행동의 이유 | 왜 이 금융 action을 하는지 | "오늘 안에 꼭", "처음 내는 거라서" |
| 수취인 관계 | 누구에게 보내는지 | "와이프한테", "아버지한테" |
| 메모 문구 | 거래 메모에 드러나는 맥락 | "생활비", "정산금", "장례비" |
| 행동의 순서·조합 | 여러 금융 행동의 패턴 | 공동통장 정리 + PDF 발급 |
| 금융 행동의 속성 | 금액 규모, 반복성, 긴급성 | "매달", "당분간", "큰돈" |

## Chatbot persona

- 유능한 은행 창구 직원처럼 행동한다.
- 한 번에 하나씩 묻는다.
- 사용자의 단서를 요약하여 Life Event를 확정하지 않는다.
- 사용자 단서를 너무 노골적으로 되받아 말하지 않는다.
- 업무 처리 중심으로 진행하고 마지막에는 처리 완료 또는 변경/해지 안내로 마무리한다.

## Dialogue template

```text
1. user: 맥락 없이 용건 투척
2. assistant/user: 정보수집, 단서 1 등장
3. assistant/user: 처리 정보 보충, 단서 2 등장
4. assistant/user: 연계 제안·수락, 선택적으로 단서 3 등장
```

## Checklist

- Life Event 직접 언급 없음
- 단서 2~3개가 서로 다른 채널에 분산됨
- 단서 하나만으로는 event 확정이 어렵고 조합해야 함
- assistant가 Life Event를 확정하거나 요약하지 않음
- 한 대화는 하나의 대표 FA action에 대응함
